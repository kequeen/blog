# go中的并发
并发其实最复杂的是变量共享的问题，golang中的解决方案说的是，不要通过共享内存去通信，而是通过通信去共享内存

并且其实就算协程的代价很低，有协程池还是更优的,并且更重要的是控制系统的负载。
比如有的场景，可能只能承受几十并发，而如果你请求端不做限制，其实很容易把下游冲垮，做这种限流，线程池其实就是一种很合适的方式。
